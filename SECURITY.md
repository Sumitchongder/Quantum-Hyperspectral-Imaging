# Security Policy

## Supported Versions
We actively maintain and provide security updates for the latest version of the Q-HSI framework.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability
**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a potential security risk (e.g., data leakage in the PCA pipeline, dependencies with known vulnerabilities, or adversarial loopholes), please report it by emailing:

**sumitchongder960@gmail.com**

Please include:
* A description of the vulnerability.
* A proof-of-concept or steps to reproduce.
* Your contact information for follow-up.

## Our Commitment
We will acknowledge your report within **48 hours** and provide a detailed response regarding the next steps within **5 business days**. We are committed to a responsible disclosure process.

## Data Privacy Note
This project uses the **HAM10000** public dataset. Users are reminded to comply with the [Harvard Dataverse terms of use](https://dataverse.harvard.edu/dataset.xhtml?persistentId=doi:10.7910/DVN/DBW86T) and ensure no Patient Health Information (PHI) is processed through non-secure quantum simulators or public clouds.****
